Undecided

Undecided<T, E>: Result<T, E> & (IsOk<T> | IsErr<E>)
  • Defined in src/result/result.ts:659

A utility type which represents a result that must be either Ok or Err. This contrasts with Result, which is neither Ok or Err as far as the compiler is concerned!

This type makes Result.isOk and Result.isErr more powerful in some situations. For example:

const thisWorks = (result: Result<number, Error>) => {
    if (result.isOk()) {
      // This works fine with the normal `Result` interface
      result.intoOk();
    }
}

const thisDoesNotWork = (result: Result<number, Error>) => {
    if (result.isErr()) {
      // The compiler knows that `result` implements `IsErr` here...
      throw new Error("result is empty");
    }

    // ...but here it is still just `Result`, which does not
    // implement `IsOk` i.e. DOES NOT COMPILE!
    result.intoOk();
}

const butDoesWithUndecided = (result: Result.Undecided<number, Error>) => {
    if (result.isNone()) {
      // The compiler knows that `result` implements `IsErr` here...
      throw new Error("result is empty");
    }

    // ...and that it must therefore implement `IsOk` here!
    result.intoOk();
}

Why don't we just use this type to represent all results? Honestly, we find that using a union with a generic makes the compiler sweat when it occurs all over the place.

As shown above, Result can still be narrowed in some situations. This type is still available for convenience but it isn't the default.

Const flatten

  • flatten<T, E>(result: Result<Result<T, E>, E>): Result<T, E>
    • Defined in src/result/result.ts:700

    Turns a result of a result into a result. In other words, it un-nests a nested result.

    Note that this only works for one level of nesting! It's easy to write code that works for any number of levels but it's not easy to express this using TypeScript.

    const result = Result.ok(Result.ok(1));
    Result.flatten(result);
    // returns Result.ok(1)
    todo

Const isResult

  • isResult<T, E>(value: any): value is Result<T, E>
    • Defined in src/result/result.ts:731

    True if the value is an result, else false.

    Implementation note: an result is defined as an instance of Ok or Err. These private classes both implement Result. For avoidance of doubt, Ok and Err are not exposed by this package.

    isResult will infer the wrapped type if possible. Note that this provides no runtime guarantee that the option is of type T.

    Result.isResult(Result.ok(1));
    // returns true

    Result.isResult(Result.err(new Error()));
    // returns true

    Result.isResult(1);
    // returns false

  • transpose<T, E>(result: Result<Option<T>, E>): Option<Result<T, E>>
    • Defined in src/result/result.ts:775

    Transposes a result of an option into an option of a result. Ok(None) will be mapped to None. Ok(Some(value)) and Err(error) will be mapped to Some(Ok(value)) and Some(Err(error)).

    Result.transpose(Result.err(someError));
    // returns Option.some(Result.err(someError));

    Result.transpose(Result.ok(Option.none()))
    // returns Option.none()

    Result.transpose(Result.ok(1));
    // returns Option.some(Result.ok(1))
